## 논리회로

논리회로는 디지털 신호 0과 1을 이용하여 정보를 처리할 수 있는 전자 회로이다. 컴퓨터, 아두이노, 스마트폰 등의 디지털 장치의 기본 구조를 이루는 핵심 기술이다. 0은 전압이 낮은 상태 Low이며, 거짓으로 취급된다. 1의 경우는 전압이 높은 상태 High이며 참으로 취급된다.

논리회로는 여러 논리 게이트를 조합하여 특정 조건에 따른 출력을 결정할 수 있다. 기본 논리 게이트로는 먼저 AND 게이트가 있다. 입력이 모두 1인 경우에만 출력이 1이며 논리식은 Y = A ⋅ B(Input : A,B Output : Y)이다. 예시로는 자동차 시동 과정이 있는데, 브레이크 밟기(1)와 기어 중립(1) 두 조건을 만족해야 시동 가능하다.
다른 기본 논리 게이트로는 OR 게이트가 있다. OR 게이트의 경우 입력 중 하나라도 1인 경우 출력이 1이 된다. 논리식은 Y = A + B 이다. 예로는 화재 경보가 있는데, 연기 감지와 온도 상승 등 하나라도 발생하는 경우 경보가 울리게 되는 논리합 원리를 가지고 있다.

## 마이크로 컨트롤러와 마이크로 프로세서의 차이점

마이크로컨트롤러(MCU)와 마이크로프로세서(MPU)는 모두 연산을 수행하는 반도체 칩이지만, 마이크로컨트롤러는 하나의 칩에 CPU와 주변 장치를 포함한 제어용 컴퓨터이며, 마이크로프로세서는 계산 중심의 고성능 CPU이기에 설계 목적과 내부 구성에서 차이가 있다.

차이점은 다음과 같이 볼 수 있다.

           마이크로프로세서(MPU)       / 마이크로컨트롤러(MCU)

 주요 목적 / 고성능 연산 처리             /    장치 제어 및 자동화
 구성      / CPU 중심                     /CPU + 메모리 + 주변장치 통합
 외부 부품 / RAM,저장장치,입출력 장치 필요 /    대부분 칩 내부에 포함
 처리 성능 / 매우 높음                     /  비교적 낮음
 전력 소비 / 높음                          /  낮음
 가격      / 비쌈                          /  저렴
 운영 방식 / 운영체제(OS) 위주             /  펌웨어 직접 실행 위주
 예        / PC,노트북,스마트폰 AP         /  아두이노,가전제품,센서 장치

이런 차이를 가지고 있는 이유는 사용 목적이 다르기 때문이다. 마이크로프로세서의 경우 복잡한 계산이 목적이기에 빠른 CPU, 큰 메모리, 높은 처리 능력이 필요하다. 그렇기에 CPU 성능을 높이고 필요한 부품은 외부에 연결하는 구조가 효율적이다.
반면 마이크로컨트롤러의 경우 단순반복적인 특정 장치 제어가 목적이기에 작은 크기, 낮은 전력 소비, 가격, 안정성 등이 중요하다. 따라서 MCU는 CPU와 주변 장치가 포함되어 있는 칩으로 구성되어 있다.

## 아두이노 칩셋

아두이노의 칩셋은 기본적으로 마이크로컨트롤러(MCU) 계열이다. 아두이노 보드에 따른 사용되는 칩셋에는 여러가지가 있다. Arduino Uno, Nano, Mega의 경우에는 8Bit의 ATmega328P 등을 사용하며 가장 널리 사용된다. 반면 Arduino Due, Zero, Portenta의 경우에는 32Bit 기반의 MCU를 사용한다.

8Bit MCU는 한 번에 8자리 데이터 처리가 가능하지만, 비트가 높아질수록 한 번에 더 많은 데이터 처리가 가능하다. 8Bit의 경우 연산 능력이 상대적으로 낮고, 동작 속도도 수~수십 MHz이다. 32Bit의 경우 연산 능력이 높고 동작 속도도 수십~수백 MHz이다. 그러나 32Bit가 무조건 좋다고는 볼 수 없다. 메모리 크기나 전력 소비, 가격에서는 8Bit MCU가 우세한 편이다. 그래서 8Bit MCU는 구조가 단순한만큼 주로 LED 제어가 버튼 입력등 간단한 처리에 사용되고, 32Bit MCU의 경우 영상 처리나 무선 통신 등 보다 복잡한 프로그램에 사용된다.

## MCU와 MPU에 사용되는 소프트웨어

앞에서 말했듯이 MCU의 경우 주로 펌웨어(Firmware)를 실행한다. 펌웨어는 MCU 내부의 Flash 메모리에 저장되어 하드웨어를 직접 제어하는 프로그램이다. 계열에 따라 사용되는 소프트웨어도 다르며, Arduino 계열은 C/C++, Arduino IDE / STM32 계열은 STM32CubeIDE, Keil MDK 등이 대표적이다. / ESP32 계열은 PlatformIO, ESP-IDF를 사용한다.
또 비트 수에 따른 소프트웨어에도 차이가 있으며, 8Bit MCU의 경우 ATmega328P등의 펌웨어로만 소프트웨어를 사용하지만, 16Bit나 32Bit의 경우 펌웨어에 RTOS까지 사용하는 경우가 많다.

마이크로프로세서의 경우에는 분야에 따른 다른 OS를 사용한다. PC는 윈도우, 리눅스, macOS / 스마트폰은 Android, iOS / 서버는 Linux , Unix / 임베디드 장비는 Embedded Linux 를 사용한다.
