## 임베디드 시스템의 필수조건
임베디드 시스템을 효율적으로 적용하기 위해서는 여러한 조건들이 필요하다. HW/SW, 저전력, 경량화, 실시간성 등이 필수조건으로 필요하며 이러한 것들의 개선을 통하여 임베디드 시스템을 더 뛰어나게 적용할 수 있을 것이다.

## 아두이노 개요
아두이노(Arduino)는 마이크로컨트롤러(MCU)를 기반으로 한 오픈소스 하드웨어/소프트웨어 플랫폼이다. 전자회로와 프로그래밍을 쉽게 배우고 다양한 제어 시스템을 만들 수 있도록 개발되었다. 주요 특징으로는 하드웨어 설계도와 소프트웨어가 공개되어 오픈소스로 누구나 사용할 수 있으며, C/C++ 기반의 아두이노 언어를 사용하며, 전용 IDE로 쉽게 개발 할 수 있다는 장점을 가지고 있다. 또한 LED, 센서, 모터, LCD, 블루투스, Wi-Fi 등 다양한 장치에 연결하여 사용할 수 있다. 확장성이 좋고 비용이 저렴한 것 동안 장점이다. 단점으로는 고성능 연산에는 적합하지 않고, 메모리 처리 속도가 제한적이며 대규모 시스템 개발에는 한계가 있다는 것이다.

## 아두이노의 최적 하드웨어
아두이노에 사용되는 마이크로컨트롤러(MCU)는 매우 다양하며, 보드 종류에 따라 다르고 성능과 기능에도 차이가 있다. 

- ATmega328P
CPU : 8bit AVR , 동작 속도 : 16 MHz , Flash 메모리 : 32KB , SRAM : 2KB
가장 보편적인 MCU이며 교육용 및 간단한 제어 프로젝트에 적합하다. 그렇기에 메모리와 연산 성능은 제한적이다.

- ATmega2560
CPU : 8bit AVR , 동작 속도 : 16 MHz , Flash 메모리 : 256KB , SRAM : 8KB
앞의 MCU보다 메모리와 입출력 핀이 크게 증가한 MCU이다. 여러 센서와 모터를 동시에 제어하는 프로젝트에 적합하다.

- ATmega32U4
CPU : 8bit AVR , 동작 속도 : 16 MHz , Flash 메모리 : 32KB , SRAM : 2.5KB
USB 기능이 내장되어 있어 별도의 USB 변환 칩이 필요가 없다는 특징을 가지고 있다. 이 특징으로 키보드나 마우스 등의 USB HID 장치 구현이 가능하다.

- SAMD21G18
CPU : 32bit AVR Cortex-M0+ , 동작 속도 :  48 MHz , Flash 메모리 : 256KB , SRAM : 32KB
32bit ARM 기반으로 처리 속도와 메모리 성능이 크게 향상된 하드웨어이다. 저전력 및 고성능이 필요한 응용에 적합하다.

- ESP32
CPU : 32bit 듀얼코어 Xtensa , 동작 속도 : Max 240 MHz , Flash 메모리 : 일반적으로 4MB(외부 Flash) , SRAM : 약 520KB
가장 높은 성능을 제공하며 Wi-Fi와 Bluetooth가 기본으로 내장되어 있어 IoT(사물인터넷) 프로젝트에 많이 사용된다. 또한 멀티테스킹과 실시간 통신에 유리한 편이다.

일반적으로 성능은 나열한 순서대로 ATmega328P < .... < ESP32 이다. 다만 ATmega2560의 경우 메모리와 입출력 핀 수는 많지만 CPU 자체의 연산 성능은  ATmega328P와 같은 8bit AVR 16MHz 수준이다. 따라서 연산 성능만 보면 SAMD216G18이 더 우수하다고 볼 수 있다.

## 아두이노에 탑재되는 소프트웨어
아두이노에 탑재되는 소프트웨어는 크게 4가지로 볼 수 있다.

1. Arduino IDE
아두이노 프로그램을 작성하고 컴파일하여 보드에 업로드하는 통합 개발 환경(IDE)이다. C/C++ 기반 아두이노 언어를 사용하며 주요 기능으로는 코드 작성 및 편집, 컴파일, 보드에 프로그램 업로드, 시리얼 모니터를 통한 데이터 확인이 있다.

2. 스캐치(Sketch)
사용자가 작성하는 응용 프로그램으로, .ino 확장자를 사용한다. 컴파일 후에는 마이크로프로세서 Flash 메모리에 저장되어 실행된다. 기본적으로 다음과 같은 함수 2개로 구성된다.
setup() : 전원이 켜질 때 한 번 실행
loop() : 계속 반복 실행

3. 부트로더(Bootloader)
MCU에 미리 저장된 부팅을 하기위한 프로그램으로 보면 된다. 아두이노 IDE를 원할하게 사용하기 위한 프로그램을 실행시키며, 부트로더가 고장날 경우에 업로드 과정에서 오류가 발생할 수 있다.

4. 라이브러리(Library)
센서, LCD, 모터, Wi-Fi, Bluetooth 등의 기능을 쉽게 사용할 수 있도록 제공되는 프로그램 모음이다. #include 명령으로 원하는 라이브러리를 불러올 수 있다.

## 저전력 시스템 및 경량화

저전력 시스템과 경량화는 임베디드시스템에서 매우 중요한 요소이다. 배터리 사용시간 연장, 발열 감소, 시스템의 신뢰성 향상, 에너지 절감 등을 위하여 전력소비를 최소화하는 저전력 시스템을 도입한다.
저전력 MCU 사용, 슬립 모드 및 절전 모드 활용, 클럭 주파수와 전압 제어, 미사용 주변장치 전원 차단, 효율적인 소프트웨어 및 알고리즘 설계 등을 통하여 저전력을 구현할 수 있다.

경량화는 시스템의 크기와 무게, 메모리 사용량 및 연산량을 줄여 효율적으로 사용하는 기술이다. 메모리 절약, 처리 속도 향상, 제조 비용 절감, 소형 기기 구현을 위해서 경량화는 임베디드시스템에서 중요한 요소이다.
프로그램 코드 최적화, 불필요한 기능 제거, 메모리 사용 최소화, 효율적인 자료구조 및 알고리즘 사용, 소형 부품 사용, 운영체제 및 라이브러리 최소화를 통해 경량화할 수 있다.

아두이노의 배터리 사용시간은 보드 종류와 연결된 센서, 동작 방식에 따라 크게 달라진다. Arduino Uno를 기준으로 전력소비량과 배터리 사용 시간을 유추해보면,
Arduino Uno의 스펙을 다음과 같이 가정한다. 
동적 전압 : 5V / 평균 소비 전류 : 50mA / 배터리 : 5V 2000mAh

 Arduino Uno의 소비 전력 : 5V * 50mA = 0.25mA 약 0.25mA로 전력소비량을 추정 할 수 있다.
 배터리 사용시간 : 2000mAh % 50mA = 40시간 40시간으로 볼 수 있다.
 여기서 슬립 모드를 사용해 평균 소비 전류를 5mA까지 줄인다면 400시간 가까이 사용할 수 있다.

## 아두이노 데이터 전송

아두이노는 센서를 이용해 데이터를 수집하고, 저장장치에 저장하거나 유무선 통신을 통하여 다른 장치로 데이터를 전송 할 수 있다.

1. 데이터 수집

 온도/습도 센서, 조도 센서, 초음파 센서, GPS 센서 등, 아닐로그/디지털 입력을 통하여 다양한 센서의 데이터를 수집한다. 

2. 데이터 저장

 수집한 데이터를 다양한 저장장치에 저장한다. 저장 방법에 따른 각 특징은 다음과 같다.
 내부 EEPROM - 소량 데이터 저장, 전원이 꺼져도 데이터 유지
 SD 카드 모듈 - 대용량 데이터 저장 가능
 Flash 메모리 - 프로그램과 일부 데이터를 저장
 외부 메모리 모음 - EEPROM , FRAM 등을 연결하여 저장 공간 확장

3. 데이터 통신

 유무선 통신을 통하여 데이터를 다른 장치로 전송한다.

 유선 통신
 USB(시리얼 통신) : PC와 연결하여 데이터 송수신
 UART : 다른 MCU와 통신
 I²C : 센서 및 주변장치와 통신
 SPI : SD 카드, LCD 등 고속 주변장치와 통신

 무선 통신
 블루투스 : 스마트폰, 데스크탑 등의 전자기기와 근거리 통신
 Wi-Fi : 인터넷 및 클라우드 서버 전송
 LoRa : 장거리 저전력 통신
 Zigbee : 저전력 무선 네트워크 구축

이러한 유무선 통신 방식으로 스마트팜, 환경 모니터링, 건강 관리 등에 데이터 전송을 활용할 수 있다.
